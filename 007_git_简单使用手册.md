# Git 实用教程 · 2026 现代个人/小团队版

> 作者：WuDiXianXin（基于社区最佳实践持续更新）  
> 目标人群：个人开发者、小团队、同时使用 GitHub + Gitee 的同学  
> 当前推荐风格：**Trunk-based development**（主干开发） + 短生命周期特性分支（最主流做法）

## 0. 为什么现在大家更倾向于 main + 短特性分支？

2023~2026 年主流趋势已经从传统的 GitFlow 转向更轻量的 **Trunk-based development** / **GitHub Flow** 风格：

- main 分支始终保持可部署状态
- 特性/修复都开短生命周期分支（通常1~4天内合并）
- 通过 Pull Request 进行 Code Review
- 频繁小步提交 & 合并，减少冲突
- CI/CD 友好

**GitFlow**（develop + release + hotfix）现在多用于有严格发布周期的大型项目或需要同时维护多个历史版本的产品。

## 一、一次就好 - 全局推荐配置（2026建议）

```bash
# 基本身份（必须）
git config --global user.name "WuDiXianXin"
git config --global user.email "your-real-email@example.com"

# 现代默认主分支（几乎所有平台默认 main）
git config --global init.defaultBranch main

# 推荐行为（大大提升体验）
git config --global core.autocrlf input          # Linux/Mac 推荐，Windows 用 true
git config --global core.quotepath false         # 正确显示中文路径
git config --global core.editor "code --wait"    # 用 VSCode 作为提交信息编辑器（或 vim/nano）
git config --global pull.rebase false            # 默认 merge（更安全），喜欢 rebase 可改 true
git config --global fetch.prune true             # 自动清理已删除的远程分支
git config --global push.autoSetupRemote true    # git push 时自动设置上游（Git 2.37+）

# 超好用：颜色 + 简洁输出
git config --global color.ui auto
```

## 二、最推荐的仓库初始化流程（个人/小项目）

```bash
mkdir awesome-project && cd awesome-project
git init

# 推荐创建的「标配三件套」
echo "# Awesome Project" > README.md
echo "## 快速开始\n\n\`\`\`bash\n# 待补充\n\`\`\`" >> README.md

# LICENSE（推荐开源协议：MIT / Apache-2.0 / GPL-3.0）
# 快速方式：
curl -L https://www.mit-license.org/ -o LICENSE

# .gitignore（强烈建议使用在线生成器）
# 推荐网站：https://www.toptal.com/developers/gitignore
# 常见组合：Node,Python,JetBrains,Windows,macOS,VSCode,venv,...
# 示例最常用几行：
cat <<EOF > .gitignore
# 通用
.DS_Store
*.log
*.tmp

# 编辑器/IDE
.vscode/
.idea/

# 构建/依赖
node_modules/
dist/
build/
EOF

# 首次提交（现代 commit message 风格）
git add .
git commit -m "chore: initial project structure with README, LICENSE, .gitignore"

# 确保主分支叫 main
git branch -M main
```

## 三、同时推 GitHub + Gitee（最省事写法）

```bash
# 方法一：最推荐 - 一个 remote 双推（以后 git push 一次搞定）
git remote add origin git@github.com:WuDiXianXin/awesome-project.git
git remote set-url --add --push origin git@gitee.com:wudixianxin/awesome-project.git

git push -u origin main   # 第一次要 -u 建立追踪关系

# 之后就这么简单：
# 日常开发 → git push  → 两边同时更新
```

## 四、现代日常开发推荐流程（Trunk-based 风格）

```text
main    ←──────────────┐
           ▲            │
           │ Pull       │
feature/xx ┼────────────┘   (PR → squash & merge 或 rebase merge)
```

常用命令组合：

```bash
# 1. 拉取最新（保持 main 干净）
git switch main
git pull

# 2. 开新特性分支（命名推荐：类型/简短描述）
git switch -c feat/add-login-page    # 功能
git switch -c fix/bug-search-input   # 修复
git switch -c refactor/user-service  # 重构

# 3. 正常开发（尽量原子提交）
git add .
git commit -m "feat: add login page basic layout"
git commit -m "fix: correct form validation logic"

# 4. 保持与 main 同步（推荐 rebase 方式，更干净历史）
git fetch
git rebase origin/main               # 冲突 → 解决 → git rebase --continue

# 5. 推分支 & 创建 PR
git push -u origin feat/add-login-page

# → 去 GitHub/Gitee 创建 Pull Request（建议 squash merge 或 rebase merge）
```

## 五、非常实用的 Git 别名（极大提升效率）

添加到 `~/.gitconfig` 的 `[alias]` 部分：

```ini
[alias]
    s     = status --short --branch
    ss    = status
    co    = checkout
    cob   = checkout -b
    br    = branch -vv
    ci    = commit
    cim   = commit --amend --no-edit
    ca    = commit --amend
    cm    = commit -m
    d     = diff
    dc    = diff --cached
    ds    = diff --staged
    lg    = log --graph --oneline --decorate --all
    ll    = log --pretty=format:"%C(yellow)%h%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset" --abbrev-commit
    last  = log -1 HEAD --stat
    undo  = reset --soft HEAD~1          # 撤销最后一次提交但保留改动
    wipe  = reset --hard HEAD            # 危险！丢弃所有未提交改动
    clean-all = !git clean -fd && git reset --hard HEAD
    sync  = !git pull && git fetch --prune
    fresh = !git fetch --prune && git rebase origin/main
```

使用示例：`git s`、`git lg`、`git cm "fix: typo"` 等

## 六、实用进阶操作速查

| 场景                              | 推荐命令组合                                      | 备注                             |
|-----------------------------------|---------------------------------------------------|----------------------------------|
| 撤销最后一次 commit（保留改动）   | `git undo` 或 `git reset --soft HEAD~1`          | 常用                             |
| 彻底丢弃工作区所有改动            | `git wipe` 或 `git reset --hard HEAD`            | 危险！先确认                     |
| 清理未跟踪文件                    | `git clean -fd`                                      | -n 先预览                        |
| 从历史彻底删除敏感文件            | `git filter-repo --path secrets.env --invert-paths` | 之后强制推送                     |
| 改写最近几次 commit 信息          | `git rebase -i HEAD~5`                               | 用 reword/edit 等                |
| 快速同步 main 并开新分支          | `git fresh && git cob feat/xxx`                      | 使用上面别名                     |
| 强制更新远程（危险）              | `git push --force-with-lease`                        | 比 --force 更安全                |

## 七、一些现代最佳实践建议（2026）

1. 尽量使用**语义化提交**（Conventional Commits）  
   `feat:`、`fix:`、`docs:`、`refactor:`、`chore:`、`perf:`、`test:` 等

2. 提交信息**用英文**（团队协作必备），中文项目可中英双语

3. PR 标题与提交信息保持一致，便于生成 CHANGELOG

4. 个人项目也建议养成**开 PR 自审**的习惯

5. 重要的个人配置建议做成 **dotfiles** 仓库并同步 GitHub + Gitee

6. 学会善用 `git stash`、`git worktree`、`git bisect`（找问题始作俑者）
